# 非功能需求延期清单

更新时间：2026-03-04  
策略：非功能需求暂缓，先保障功能闭环；本清单用于后续补齐追踪。

---

## 1. 暂缓原则

1. 当前阶段优先完成功能需求与前后端业务链路闭环。
2. 非功能项仅保留最低限度保障（可运行、可测试、可回滚基础）。
3. 非功能深度建设在功能闭环后集中推进。

---

## 2. 延期项（按 NFR 分类）

## 2.1 性能（NFR-PERF）

1. `NFR-PERF-001`：聊天接口 P95 SLA 达标压测与调优（延期）。
2. `NFR-PERF-002`：文档入库并行优化与批任务吞吐调优（延期）。
3. `NFR-PERF-003`：检索与工具调度长尾优化（延期）。

## 2.2 可靠性（NFR-REL）

1. `NFR-REL-001`：外部数据源高可用切换策略完善（延期）。
2. `NFR-REL-002`：关键任务幂等与重试策略全面收敛（延期）。
3. `NFR-REL-003`：优化任务完整灰度发布体系（延期）。

## 2.3 可观测（NFR-OBS）

1. `NFR-OBS-001`：端到端全链路 trace 一致性增强（延期）。
2. `NFR-OBS-002`：指标体系完善（检索质量、反馈分布等）与仪表盘建设（延期）。
3. `NFR-OBS-003`：统一错误上下文体系扩展（延期）。
4. `NFR-OBS-004`：流程版本/Prompt/执行轨迹联查增强（延期）。

## 2.4 安全与隐私（NFR-SEC）

1. `NFR-SEC-001`：密钥管理强化（轮转、权限隔离）延期。
2. `NFR-SEC-002`：文档与会话保留周期、清理策略产品化延期。
3. `NFR-SEC-003`：高风险优化人工审批能力细化延期。

## 2.5 可维护性（NFR-MTN）

1. `NFR-MTN-002`：Milvus/Chroma 全量切换验证延期（当前默认 Chroma）。
2. `NFR-MTN-003`：需求->设计->测试->发布自动化追踪完善延期。
3. `NFR-MTN-005`：Prompt 多环境隔离治理深化延期。

---

## 3. 当前最低保障（已保留）

1. 可运行：后端主链路与关键 API 可用。
2. 可验证：单测、语法检查、flake8 持续执行。
3. 可追踪：基础迭代记录与 CHANGELOG 持续维护。

---

## 4. 恢复补齐触发条件

满足以下条件后，切换到非功能补齐阶段：

1. 前端五大页面完成并联调通过。
2. MVP 功能性需求清单达到“可验收”状态。
3. Agent 编排与流程治理关键缺口补齐。
