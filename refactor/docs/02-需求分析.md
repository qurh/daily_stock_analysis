## 1. 背景与目标

现有系统已具备股票分析、报告生成、回测、通知、多入口（CLI/API/Web/Bot）等能力，但核心能力仍以“单次分析任务”为主，缺乏系统级学习闭环、长期记忆、多源宏观风险融合和智能工具调度能力。

本轮新需求目标是将系统升级为：

1. 具备知识学习与自我进化能力的智能股票助手。
2. 具备多轮对话与长期记忆能力的 RAG 系统。
3. 具备文档优化、一键导入、知识可检索与可引用能力。
4. 具备技术面 + 宏观 + 信用 + 舆情的综合分析能力。
5. 具备模块化架构与可插拔工具体系。
6. 具备反馈驱动优化与持续迭代能力（含回测闭环）。
7. 具备“对话核心认知沉淀 -> 策略提炼 -> 策略反哺”的自我进化路径。

## 2. 已确认约束与原则

### 2.1 业务约束

- 现有功能基线默认全保留（见 `refactor/docs/01-全量保留功能清单.md`）。
- 通知渠道全部保留，不裁剪，后续通过插件化改造提升扩展性。
- 回测能力为核心能力，必须纳入 MVP（历史口径“F31/F32 回测能力”要求保持生效）。

### 2.2 工程原则

- 兼容优先：保留现有 `.env` 配置语义，新增配置向后兼容。
- 可观测优先：所有核心链路需可追踪（日志、指标、审计、错误码）。
- 安全优先：密钥、敏感配置、用户数据和反馈数据需可控存储与访问隔离。
- 可验证优先：需求应可通过测试与指标验收，不依赖主观描述。

## 3. 角色与核心场景

## 3.1 角色定义

- `个人投资者（主用户）`：导入知识、提问、查看分析报告、给出反馈。
- `系统维护者（你）`：配置数据源、模型、策略开关，查看系统运行质量。
- `平台服务（系统内部）`：执行分析、检索记忆、管理知识库、触发回测与优化。

## 3.2 核心业务场景

1. 用户上传一批 Markdown 研究笔记，系统自动优化并入库。
2. 用户在多轮对话中询问“某股当前风险和建议”，系统结合长期记忆与最新数据回复。
3. 用户通过 chatbot 的长期交流逐步形成“核心认知”，系统自动沉淀为知识库 MD 文档，并支持用户审核修订。
4. 系统基于知识库文档提炼“分析策略”和“交易策略”，用于后续分析与策略优化。
5. 用户可按目标选择和编排分析子流程（如技术面 -> 宏观 -> 风险汇总），并为每个子流程配置 Prompt 模板。
6. 用户发起个股/大盘分析，系统综合技术面、宏观、信用、舆情生成决策仪表盘。
7. 用户对分析结果点赞/差评并打标签，系统用于后续提示词与策略优化。
8. 系统定期回测历史建议并输出效果，反馈给优化模块形成闭环。

## 4. 现状能力与新增需求差距分析

| 需求主题 | 当前能力现状 | 差距结论 |
|---|---|---|
| 知识学习与自我进化 | 有搜索与分析，无结构化知识库闭环，缺少对话认知沉淀与策略提炼链路 | 需新增文档处理、向量库、认知沉淀与策略提炼引擎 |
| 多轮对话与长期记忆 | 有分析 API/报告，无记忆型对话服务 | 需新增 Chat+Memory+RAG 统一服务 |
| MD 文档优化与一键导入 | 暂无文档导入与优化管线 | 需新增 ingestion pipeline 与 UI 入口 |
| 股票分析扩展 | 已有技术面、舆情、回测 | 需新增宏观/信用数据域与综合风险模型 |
| 模块化与工具集成 | 现有模块偏单体耦合 | 需重构为模块化边界 + 工具编排层 |
| 分析流程编排与 Prompt 治理 | 缺少“子流程级”可编排与可配置 Prompt 能力 | 需新增流程模板、节点编排、Prompt 版本治理与回滚能力 |
| 反馈机制与自我优化 | 已有回测，但与提示词优化弱耦合 | 需新增反馈采集、策略版本化、周期优化任务 |

## 5. 范围定义（需求范围）

## 5.1 In Scope（本次需求纳入）

- 知识库构建：文档导入、清洗、分块、摘要、向量化、检索。
- RAG With Memory：短期会话记忆 + 长期向量记忆 + 历史摘要压缩。
- 聊天接口：支持多轮上下文引用与知识库引用链路。
- 对话认知沉淀：从 chatbot 交互中提炼核心认知，沉淀为可审核的知识库 MD 文档。
- 策略提炼：从知识库文档抽取分析策略与交易策略，并作为自我优化输入。
- 分析流程编排：支持子流程模块化定义、任意编排、节点级 Prompt 定制与版本管理。
- 分析能力扩展：技术面增强（含 TA-Lib 指标扩展）、宏观数据、信用风险数据、新闻情绪。
- 智能 Agent 工具调度：行情、新闻、宏观、信用、知识库检索工具。
- 回测与反馈闭环：反馈采集、回测结果、提示词与策略优化机制。
- 全渠道通知保留并插件化改造要求落地到需求层。

## 5.2 Out of Scope（本阶段不纳入）

- 自动交易下单、券商 API 实盘执行。
- 资产组合自动调仓与资金管理执行器。
- 面向多租户 SaaS 的计费与租户隔离系统。
- 超低延迟高频交易级别能力。

## 7. 非功能需求（Non-Functional Requirements）

## 7.1 性能

- NFR-PERF-001：聊天接口 P95 响应时间在常规负载下可控（需按部署规格定义基线，建议 < 8s）。
- NFR-PERF-002：文档入库任务支持异步化与并行处理，单文档失败不阻塞批任务。
- NFR-PERF-003：向量检索与工具调度需具备超时与降级策略，避免长尾阻塞。

## 7.2 可用性与可靠性

- NFR-REL-001：外部数据源故障时应自动切换/降级，不得导致系统整体不可用。
- NFR-REL-002：关键任务（分析、回测、入库）需支持重试与幂等处理。
- NFR-REL-003：优化任务应具备安全闸门（离线评估/灰度/回滚）。

## 7.3 可观测性

- NFR-OBS-001：必须提供端到端 trace（请求 -> 检索 -> 工具 -> 生成 -> 存储）。
- NFR-OBS-002：核心指标至少包含：调用成功率、延迟、检索命中率、回测指标、反馈分布。
- NFR-OBS-003：错误需结构化记录（错误码、模块、外部依赖、重试次数）。
- NFR-OBS-004：流程编排版本、节点执行轨迹、Prompt 版本需可查询并可关联到单次分析结果。

## 7.4 安全与隐私

- NFR-SEC-001：API Key、Token、密钥必须安全存储，不进入日志明文。
- NFR-SEC-002：用户上传文档与对话内容可配置保留周期与删除策略。
- NFR-SEC-003：高风险自我优化能力需可人工审批（可关闭自动生效）。

## 7.5 可维护性与扩展性

- NFR-MTN-001：模块边界清晰，数据采集、知识库、分析生成、交互接口解耦。
- NFR-MTN-002：工具、模型、向量库需支持替换（Milvus/Chroma 可切换）。
- NFR-MTN-003：需求到实现保持可追踪（需求ID -> 设计 -> 测试 -> 发布）。
- NFR-MTN-004：分析流程需配置化编排，新增/替换子流程节点不应修改核心编排引擎。
- NFR-MTN-005：Prompt 模板治理需支持热更新、版本回滚与环境隔离（dev/staging/prod）。

## 8. 数据与模型需求

## 8.1 核心数据对象

- `KnowledgeDocument`：原文、优化文、来源、标签、版本。
- `KnowledgeChunk`：块文本、摘要、embedding、元数据（章节路径、时间戳）。
- `ConversationSession`：会话主记录、配置策略、状态。
- `ConversationMessage`：消息内容、角色、token统计、引用证据。
- `MemorySummary`：历史对话摘要、覆盖范围、版本。
- `CognitionMemo`：从对话提炼出的核心认知 MD 草稿、审核状态、来源会话映射。
- `StrategyArtifact`：分析策略/交易策略产物、版本、来源文档集合、评估结果。
- `FlowTemplate`：分析流程模板定义（节点、连线、条件分支、默认参数）。
- `FlowNode`：子流程节点实例（节点类型、输入输出契约、执行状态）。
- `PromptTemplate`：节点级提示词模板（变量、约束、输出 schema、版本）。
- `OrchestrationPolicy`：编排策略与生效范围（任务类型、策略版本、灰度规则）。
- `AnalysisRequest/Result`：分析输入输出、指标快照、报告JSON。
- `MacroSnapshot`：宏观指标快照（GDP、就业等）。
- `CreditRiskSnapshot`：CDS、利差等风险特征快照。
- `SentimentSnapshot`：舆情情绪聚合特征。
- `BacktestResult/Summary`：回测明细与汇总指标。
- `FeedbackRecord`：评分、标签、评论、关联对象。
- `PromptVersion/StrategyVersion`：策略版本、评估结果、是否生效。

## 8.2 数据生命周期要求

- 支持数据分层：热数据（近期会话/任务）与冷数据（长期知识与历史档案）。
- 支持索引重建与向量重嵌入（embedding 模型升级时）。
- 支持审计追踪：关键优化动作需记录“谁在何时以何依据变更了什么”。

## 9. 接口与集成需求

## 9.1 内部接口

- 聊天接口：会话管理、RAG 检索、记忆策略、引用证据返回。
- 文档入库接口：上传、清洗预览、确认导入、状态查询。
- 认知沉淀接口：会话核心认知提炼、审核、入库。
- 策略提炼接口：分析策略/交易策略抽取、版本管理、生效治理。
- 流程编排接口：流程模板定义、节点编排、策略选择、版本切换。
- Prompt 管理接口：节点级 Prompt 模板创建、测试、发布、回滚。
- 分析接口：个股/大盘分析触发、结构化报告查询。
- 反馈接口：用户评分/标签提交与查询。
- 优化接口：策略版本管理、评估结果查看、启停与回滚。
- 回测接口：执行、结果、汇总、对比。

## 9.2 外部依赖（需求层）

- LLM 编排：LangChain/LlamaIndex（至少支持其一并预留另一方适配）。
- 向量库：Milvus/Chroma（二选一落地，另一方保留适配接口）。
- 市场/新闻/宏观/信用数据源：需支持多源容灾和统一模型映射。

## 10. 验收与成功指标

## 10.1 功能验收

- 文档从上传到入库全链路可用，且可在对话中被检索引用。
- 多轮对话具备上下文一致性，旧对话自动摘要，不出现上下文失控增长。
- 对话核心认知可自动沉淀为知识库 MD，并支持审核后入库。
- 系统可从知识库文档提炼分析策略和交易策略，并可被分析链路实际引用。
- 分析流程可按模块任意编排，且每个子流程 Prompt 支持独立定制与生效验证。
- 新分析报告同时包含技术面、宏观、信用、情绪维度。
- 回测链路可消费新报告并输出稳定指标。
- 反馈能够被收集并进入优化任务输入。
- 通知全渠道能力保留，新增渠道可插拔接入。

## 10.2 结果指标（建议）

- 检索命中质量：引用证据覆盖率、用户主观相关性评分。
- 分析稳定性：任务成功率、异常率、平均重试次数。
- 决策有效性：方向正确率、命中率、策略收益对比基线。
- 用户体验：会话满意度、反馈正向率、功能使用频次。

## 11. 风险与缓解

| 风险 | 影响 | 缓解策略 |
|---|---|---|
| 自动优化导致策略劣化 | 分析质量下降 | 强制离线评估 + 灰度开关 + 快速回滚 |
| 外部数据源质量波动 | 分析偏差与不稳定 | 多源交叉校验 + 数据质量评分 + 降级策略 |
| 对话记忆膨胀 | 成本与延迟上升 | 摘要压缩 + 长短期记忆分层 + token预算 |
| 向量库/框架替换成本高 | 架构锁死 | 抽象仓储接口 + Provider 适配层 |
| 金融场景误用风险 | 用户误解为投资建议 | 明确免责声明 + 风险提示模板强化 |
