# 功能需求完成度复盘

更新时间：2026-03-04  
复盘范围：`refactor/docs/02-需求分析.md` 中 FR 需求、`refactor/docs/06-MVP实施方案.md` 的 MVP In Scope  
复盘对象：`refactor/backend` 与 `refactor/frontend` 当前实现

---

## 1. 结论摘要

1. 后端核心功能主链路已基本打通，尤其是知识导入、聊天记忆、分析、回测、反馈优化、通知、策略治理。
2. 仍有多项“功能性需求”处于部分完成状态，主要集中在 Agent 工具编排、流程治理深度能力、部分高级策略能力。
3. 前端目前仅完成 M0 脚手架，MVP 要求的五大页面尚未实现，已成为当前功能交付的最大缺口。

## 3. 部分完成功能（有实现，但未达到需求完整度）

1. `FR-KB-005`：已有删除与检索，但缺少按标签/时间过滤管理、重建索引管理接口。
2. `FR-CHAT-003`：支持手动摘要接口，但未实现按阈值自动摘要触发。
3. `FR-CHAT-005`：有 `memory_policy` 字段，但缺少保留条数/阈值/过期等完整策略配置能力。
4. `FR-ANL-001`：技术分析增强已做，但 TA-Lib 指标集成尚未落地成正式能力。
5. `FR-ORC-002`：条件分支编排尚未完整落地（当前重点在串行/并行）。
6. `FR-ORC-004`（Flow 侧）：流程版本管理/差异/回滚能力不完整（Prompt 已有，Flow 仍偏配置化）。
7. `FR-ORC-005~006`：已存在策略发布闸门与回滚，但“评估->灰度->全量”通用治理流程仍不完整。
8. `FR-FDB-005`：策略发布有 backtest gate，但 Prompt/Flow 变更的统一灰度治理未闭环。
9. `FR-STR-002`：`trading` 类型提炼接口存在，但策略规则深度与交易域细化能力仍是基础版。
10. `FR-BT-004`：回测 API 已可用，但 Web 视图尚未实现。
11. `FR-NTF-003`：已有基础渠道格式化与截断，尚未形成完整“渠道 Markdown 方言策略”体系。

## 5. MVP 功能性达成判断（截至 2026-03-04）

1. 后端功能已覆盖 MVP 大部分主链路。
2. 功能性未达成项主要在：
   - Agent 编排域（FR-AGT）
   - Flow 治理深度能力（FR-ORC 部分）
   - 前端五大页面（M4 关键要求）
3. 因此前后端整体“功能性 MVP 完成定义”尚未达成，前端是当前最大阻塞项。

---

## 6. 下一阶段建议（功能优先）

1. 先完成前端五大页面首版与后端联调，满足 MVP M4 页面交付要求。
2. 前端完成后再补 Agent 工具编排能力，打通 FR-AGT。
3. 最后补流程治理深水区（Flow 版本化、灰度发布、通用评估闸门）。
