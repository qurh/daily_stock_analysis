## 1. 本迭代目标

1. 为 promtool 远端 checksum 校验增加本地元数据缓存能力。
2. 在缓存有效期内避免重复远端拉取，降低外网波动对校验链路的影响。
3. 在缓存过期时自动刷新，保持校验结果可追溯与最新性平衡。

## 2. 计划范围（Plan）

1. 先补失败测试，定义 fresh cache 与 stale cache 的预期行为。
2. 增强 `validate-promtool-installer-config.sh` 支持缓存命中与缓存刷新。
3. 同步版本、README、CHANGELOG 并完成回归。

## 3. 实际完成（Done）

1. 校验脚本增强：
   - `refactor/backend/scripts/validate-promtool-installer-config.sh`
   - 新增缓存参数：
     - `PROMTOOL_REMOTE_FETCH_CACHE_FILE`（可选）
     - `PROMTOOL_REMOTE_FETCH_CACHE_TTL_SECONDS`（默认 `3600`）
   - 当缓存文件 + 元数据时间戳在 TTL 内时，直接命中缓存并跳过远端拉取。
   - 当缓存过期时输出 stale 提示并执行远端刷新；刷新成功后更新缓存与 `.meta` 时间戳。
2. 测试增强：
   - `refactor/backend/tests/unit/test_ci_prometheus_rules_check.py`
   - 新增用例：
     - fresh cache 命中时不触发 curl 远端拉取。
     - stale cache 场景触发远端刷新并更新缓存内容。
3. 文档与版本：
   - `refactor/backend/src/app/main.py` 版本升级：`0.3.80-m3-promtool-remote-metadata-cache`
   - `refactor/backend/README.md` 增加缓存配置说明。

1. 风险描述：缓存为本地文件，默认不跨 CI 运行共享。
2. 影响范围：跨 run 仍可能受外网波动影响。
3. 缓解措施：后续可结合 GitHub Actions cache/artifact 做跨 run 复用。

## 8. 关键决策

1. 决策内容：缓存能力以可选参数方式提供，不改变现有默认严格校验行为。
2. 决策原因：保持默认链路简单可控，同时给 CI 稳定性优化预留能力。
3. 影响模块：promtool 配置治理与校验性能。
