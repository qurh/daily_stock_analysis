## 7. 风险与问题

1. 风险描述：`--json-errors` 下参数错误不再输出完整 argparse usage 文本到 stderr。
2. 影响范围：依赖 usage 文本做解析的自动化脚本。
3. 缓解措施：在 JSON context 中保留 `argv`、`exit_code`，并保持 plain 模式行为不变。

## 8. 关键决策

1. 决策内容：仅在 `--json-errors` 场景结构化捕获 parse-time `SystemExit`。
2. 决策原因：优先保证机器可消费一致性，同时不影响普通 CLI 用户体验。
3. 影响模块：sync 脚本参数解析入口。
