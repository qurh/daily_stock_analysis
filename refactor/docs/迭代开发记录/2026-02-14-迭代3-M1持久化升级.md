## 7. 风险与问题

1. 风险描述：当前队列消费与 API 同进程，吞吐与隔离性有限。
2. 影响范围：高并发下 API 延迟与任务处理可能互相影响。
3. 缓解措施：M2 前引入独立 worker 进程，消费与 API 分离。

## 8. 关键决策

1. 决策内容：优先采用 SQLite + DB Queue 完成持久化闭环。
2. 决策原因：无需外部依赖即可快速建立可测试的持久化基线。
3. 影响模块：Workflow Engine、Analysis Service、Prompt Center、Application Bootstrap。
