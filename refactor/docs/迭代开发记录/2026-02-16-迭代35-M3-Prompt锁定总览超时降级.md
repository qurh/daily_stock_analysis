## 1. 本迭代目标

1. 为 `/api/v2/prompt-lock/overview` 并行聚合增加模块级超时降级能力。
2. 避免单模块超时拖垮总览接口整体可用性。
3. 保持向后兼容并确保测试覆盖。

## 2. 计划范围（Plan）

1. 扩展 `PromptLockAuditService` 支持并行模块超时参数。
2. 并行聚合中增加 timeout/exception 降级处理与错误元数据。
3. 降级响应不进入缓存，避免污染缓存命中结果。
4. 接入运行时配置并更新文档。

## 3. 实际完成（Done）

1. 服务层改造：
   - 新增参数：`overview_module_timeout_sec`
   - 并行聚合支持模块级 timeout 降级
   - 增加响应字段：
     - `degraded`
     - `module_errors[]`
   - timeout/exception 模块返回空结构，其他模块正常返回
   - 降级响应不缓存，正常响应仍按缓存策略处理
2. 配置接入：
   - 新增环境变量：
     - `PROMPT_LOCK_OVERVIEW_MODULE_TIMEOUT_SEC`
   - 通过 `settings -> create_app -> PromptLockAuditService` 注入
3. 新增并通过测试：
   - `test_prompt_lock_overview_parallel_timeout_degrades_and_skips_cache`
   - `test_settings_reads_non_prefixed_env_vars`（新增模块超时配置断言）
4. 应用版本升级：
   - `0.3.20-m3-prompt-lock-overview-timeout-degrade`

## 7. 风险与问题

1. 风险描述：超时阈值配置不合理会导致频繁降级或无效保护。
2. 影响范围：Prompt Lock overview 聚合结果完整性与响应时延。
3. 缓解措施：后续加入命中率/降级率指标并基于流量调优阈值。

## 8. 关键决策

1. 决策内容：仅在并行多模块路径启用 timeout 降级；单模块路径保持原行为。
2. 决策原因：避免改变单模块接口语义，降低行为回归风险。
3. 影响模块：Prompt Lock overview 并行聚合流程与缓存流程。
