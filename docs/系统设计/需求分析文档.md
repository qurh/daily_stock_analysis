## 1. 项目概述

### 1.1 项目背景

本项目旨在构建一个**AI 驱动的个人投资助手与知识管理系统**。该系统能够：

- 自动化搜集市场信息与新闻情报
- 深度研究与分析投资标的
- 管理与积累投资知识资产
- 从历史数据中提炼分析策略与交易策略
- 实时盯盘并提供交易建议
- 每日复盘总结，实现系统的自我进化

### 1.2 项目定位

| 阶段 | 定位 | 说明 |
|-----|------|------|
| 初期 | 个人投资助手 | 主要服务于个人投资决策 |
| 中期 | 知识管理系统 | 侧重投资知识的研究、整理与积累 |
| 后期 | 可扩展 SaaS | 可分享给朋友，最终发展为通用平台 |

### 1.3 核心价值

1. **信息整合**：自动聚合多源市场数据、新闻、舆情
2. **AI 决策辅助**：基于知识库的 RAG 对话，提供个性化策略
3. **知识沉淀**：结构化积累投资研究成果，形成可复用的知识资产
4. **自动化盯盘**：7x24 小时监控市场，实时推送告警
5. **持续进化**：每日复盘，从实践中学习与优化

## 2. 用户需求

### 2.1 用户画像

| 属性 | 描述 |
|-----|------|
| 角色 | 个人投资者/资深散户 |
| 投资经验 | 有一定投资经验，希望借助 AI 提升效率 |
| 技术能力 | 能使用基本的管理界面，核心交互通过对话完成 |
| 核心诉求 | 减少盯盘时间、积累投资知识、获得 AI 辅助决策 |

### 2.2 用户故事

#### 故事 1：AI 驱动的知识生成

```
作为投资者，
我希望 AI 能够自主生成研究报告和分析文档，
这样我只需要提供修改意见，而不需要从头编写。

验收标准：
- AI 能够根据 Prompt 生成结构化的分析文档
- 支持对话式修改和完善
- 生成的文档可以一键保存到知识库
```

#### 故事 2：知识资产管理

```
作为投资者，
我希望有一个 Notion 风格的知识库来管理我的投资研究，
并且支持语义搜索和知识图谱关联。

验收标准：
- 支持 Notion 风格的富文本编辑
- 支持 Markdown 导入导出
- 支持按公司、行业、概念等维度分类
- 支持语义搜索（向量检索）
- 支持实体关系图谱展示
```

#### 故事 3：AI 对话助手

```
作为投资者，
我希望能够通过对话的方式与系统交互，
获取股票分析、市场观点、策略建议。

验收标准：
- 支持多轮对话和上下文记忆
- 对话过程中可以实时检索知识库
- 支持指定分析特定股票或市场
- 支持流式输出，响应速度快
```

#### 故事 4：持仓管理与盯盘

```
作为投资者，
我希望系统能够管理我的持仓，并在关键价位提醒我，
结合市场舆论和深度研究结果给出建议。

验收标准：
- 支持录入持仓（成本价、数量）
- 实时显示盈亏状况
- 支持设置价格告警阈值
- 告警时结合知识库给出分析建议
```

#### 故事 5：每日复盘与自我进化

```
作为投资者，
我希望系统每天收盘后能够自动复盘，
总结当日交易和教训，并更新知识库。

验收标准：
- 每日自动生成复盘报告
- 从复盘中提炼经验和策略
- 策略可验证、可追溯
- 知识库持续积累和更新
```

#### 故事 6：灵活配置 AI 模型

```
作为投资者，
我希望能够灵活配置和切换不同的 AI 模型，
根据任务类型选择最适合的模型。

验收标准：
- 支持多模型接入（Gemini、OpenAI、豆包、通义、文心等）
- 支持配置模型参数
- 支持任务路由（快思考用便宜模型，深度分析用高级模型）
- 可追踪模型使用成本
```

## 5. 技术约束

### 5.1 技术选型约束

| 层级 | 选择 | 说明 |
|-----|------|------|
| 前端框架 | React + Next.js | 主流生态，AI 应用友好 |
| 编辑器 | TipTap | Notion 风格，扩展性强 |
| UI 组件 | Ant Design / Shadcn | 快速开发 |
| 后端框架 | FastAPI | 高性能，异步支持 |
| 数据库 | PostgreSQL + pgvector | 业务数据 + 向量检索 |
| ORM | SQLAlchemy | 成熟稳定 |
| 消息队列 | Redis (可选) | 任务队列、缓存 |
| 部署 | Docker | 容器化部署 |
