# MVP 重启设计文档（A 股）

**项目名称：** daily_stock_analysis  
**版本：** v1.0  
**日期：** 2026-02-04  
**状态：** 设计定稿  

---

## 1. 目标与范围

### 1.1 MVP 目标
- 聚焦 A 股市场，提供**盘中分钟级刷新**与**收盘复盘报告**两条主链路。
- 交付两种产物：**Web Dashboard** + **Markdown 报告**。
- 支持**自选股维护与分组管理**，并在行情列表中提供**一键个股分析**入口。

### 1.2 非目标（本期不做）
- 持仓管理、盯盘告警、知识库、AI 对话系统。

---

## 2. 核心用户流程

1) 进入大盘概览 → 查看指数趋势与市场强弱  
2) 进入行情列表 → 分组过滤/排序 → 点击“分析”进入个股详情  
3) 维护自选股与分组（观察/趋势/高风险等）  
4) 点击“分析”触发生成 → 自动跳转到该股**最新分析 Markdown 详情**  
5) 在列表进入该股**历史报告列表**，查看所有过往分析  
6) 在全局**报告列表**查看全部历史报告  
7) 收盘后阅读/下载 Markdown 报告

---

## 3. 技术与架构

### 3.1 技术栈
- 前端：Next.js **16.1.6** + Tailwind CSS（纯 UI 层）  
- 后端：FastAPI（新服务）  
- 数据与分析：复用现有 Python 管线模块（逐步替换）
- 目录结构：遵循 `docs/架构设计详细分析报告.md` 第 6 节

### 3.2 模块复用策略（先复用再迁移）
- 数据源：`data_provider/`（策略切换与标准化）  
- LLM 分析：`analyzer.py`（结构化分析 + 摘要）  
- 新闻搜索：`search_service.py`  
- 持久化：`storage.py`（SQLite）  
- 通知：`notification.py`（可选）

### 3.3 系统形态
- 新后端提供统一 API 层，前端仅消费 API。  
- 盘中通路：分钟级拉取 → 指标计算 → 缓存/落库 → API 查询。  
- 收盘通路：当日数据聚合 → 结构化摘要 → LLM 摘要 → 报告归档。  

---

## 4. 数据流与接口

### 4.1 数据流
- 行情列表/个股详情：优先读缓存，未命中触发拉取与指标计算。  
- 个股分析：结构化指标 + 新闻摘要 → LLM 生成分析摘要 → 结果缓存。  
- 报告：收盘定时任务生成 Markdown 并归档。
- 交互规则：点击“分析”**先生成最新报告** → 生成完成即跳转到最新分析详情。

### 4.2 API 草案（最小集合）
- `GET /market/overview`  
- `GET /market/list`  
- `GET /stock/{code}/detail`  
- `POST /stock/{code}/analyze`  
- `GET /stock/{code}/reports`  
- `GET /stock/{code}/reports/latest`  
- `GET /reports`  
- `GET /reports/{id}`  
- `GET /watchlists`  
- `POST /watchlists`  
- `PUT /watchlists/{id}`  
- `POST /watchlists/{id}/stocks`  
- `DELETE /watchlists/{id}/stocks/{code}`

## 5. UI 设计规范（MVP）

### 5.1 视觉风格
深色专业金融盘面风格，低炫光，高对比度。  
涨跌色遵循 A 股习惯：红涨绿跌。  
主交互色：蓝系，强调色：琥珀。  
推荐字体：Fira Sans（正文/标题）+ Fira Code（数值/代码）。

### 5.2 关键组件
- **分组标签条**（可横向滚动，显示分组名称与当日涨跌统计，含“+”新增分组入口）  
- **行情表格（中密度 12 列）**  
  1) 代码/名称  
  2) 现价  
  3) 涨跌额  
  4) 涨跌幅  
  5) 成交量  
  6) 成交额  
  7) 换手率  
  8) 量比  
  9) 主力净流入  
  10) 市值  
  11) 行业  
  12) 操作（分析按钮 + 历史报告入口）
 - **个股报告列表页**（展示该股所有历史分析，默认时间倒序）
 - **全局报告列表页**（展示全市场所有历史分析，支持筛选）

### 5.3 交互与可达性
- 行悬停高亮，焦点态可见。  
- 按钮触达尺寸 ≥ 44px。  
- 移动端采用横向滚动 + 列选择降级。

## 6. 报告输出（Markdown）

报告结构固定，内容由结构化摘要 + LLM 生成，失败时模板降级：  
1) 大盘复盘  
2) 热点板块  
3) TopN 个股  
4) 异常波动与风险提示  
5) 明日关注要点  

---

## 7. 测试与质量

- 单元测试：数据源适配、指标计算、字段标准化。  
- 容错测试：主数据源失败自动切换。  
- 集成测试：行情列表与个股分析接口返回完整字段。  
- 报告测试：结构化摘要快照（golden files）+ LLM 结构验证。  
- E2E：行情列表 → 个股详情 → 报告查看。
