# 2026-02-06 从0到1需求分析

**状态：** 生效  
**负责人：** 项目组  
**最后更新：** 2026-02-06

---

## 1. 问题定义

当前项目从单脚本分析工具演进为“前后端分离 + API 驱动”的产品形态，但文档长期存在以下问题：
- 需求范围混杂（MVP 与长期目标混写）。
- 设计口径冲突（接口、数据模型、技术栈描述不一致）。
- 实施状态缺乏统一基线（已实现和待实现边界不清）。

从0到1阶段的首要目标不是“功能最多”，而是“最小闭环可交付”。

---

## 2. 用户与核心场景

### 2.1 目标用户

- 个人投资者：希望快速查看自选股并获得结构化分析。
- 研究型用户：希望追踪历史报告，进行对比与复盘。

### 2.2 核心场景

1. 管理股票基础信息和分组。
2. 在分组维度查看股票并触发分析。
3. 查看个股历史报告和全局历史报告。

---

## 3. MVP 范围与非目标

### 3.1 MVP 范围（必须交付）

- REQ-001：股票基础信息查询与同步。
- REQ-002：自选分组管理与分组股票维护。
- REQ-003：历史报告列表与详情读取。
- REQ-004：鉴权与用户隔离（第二批次必须补齐）。
- REQ-005：行情列表与个股详情。
- REQ-006：个股分析触发并生成报告。

### 3.2 非目标（本阶段不纳入验收）

- AI 对话系统
- 知识库/RAG
- 持仓管理
- 盯盘告警
- 策略系统

---

## 4. 功能需求拆解

| ID | 功能 | 输入 | 输出 | 验收方式 |
|---|---|---|---|---|
| REQ-001 | 股票基础信息 | 股票代码/筛选参数 | 股票列表 | API 测试 + 字段校验 |
| REQ-002 | 自选分组 | 分组名/股票代码 | 分组及成分股关系 | API 测试 + CRUD 场景 |
| REQ-003 | 历史报告 | 股票代码/报告ID | 报告列表与正文 | API 测试 + 文件读取校验 |
| REQ-004 | 鉴权与隔离 | 用户名密码/Token | 访问令牌与权限边界 | 鉴权集成测试 |
| REQ-005 | 行情与详情 | 分页排序/股票代码 | 行情字段、详情摘要 | 集成测试 |
| REQ-006 | 分析生成 | 股票代码 | 报告ID与元信息 | 流程测试（生成->查询） |

---

## 5. 非功能需求

- 一致性：统一响应结构 `{ code, message, data, request_id }`。
- 安全性：报告读取需校验用户归属；路径访问受控。
- 可追踪：需求必须映射到设计、计划、实现和测试。
- 可演进：允许当前态与目标态并存，但必须记录差异。

---

## 6. 成功指标（MVP）

- 核心 API 可用率：> 99%（开发环境联调期按接口测试通过率替代）。
- P0 需求覆盖率：100%（REQ-001~REQ-006）。
- 文档一致性：需求、设计、实施计划之间无冲突描述。
- 测试覆盖：每个 P0 接口至少包含 1 个成功场景和 1 个失败场景。

---

## 7. 约束与依赖

- 当前数据库为 SQLite，后续迁移 PostgreSQL 需单独排期。
- 现有分析管线与新 API 层并行，需控制重复数据模型风险。
- 前端业务页未落地，接口定义需保持稳定。
