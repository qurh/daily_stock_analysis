# 2026-02-06 从0到1需求分析

**状态：** 生效  
**负责人：** 项目组  
**最后更新：** 2026-02-06

---

## 1. 业务目标

将当前项目从“脚本驱动分析”升级为“可联调的产品化系统”，先完成 MVP 闭环，再沿目标态逐步扩展：
- 用户可管理股票与分组。
- 用户可查看与生成分析报告。
- 系统具备核心安全边界。
- 架构预留后续智能化与交易辅助能力。

---

## 2. 目标用户与核心任务

### 2.1 目标用户

- 个人投资者：关注日常自选股跟踪。
- 研究型用户：关注历史报告复盘与对比。

### 2.2 核心任务

1. 查询并维护股票基础信息。
2. 维护自选分组与分组股票。
3. 触发分析并查看历史报告。

---

## 3. 整体需求全景（产品级）

| 需求域 | 关键需求 | 目标阶段 |
|---|---|---|
| 数据与分组基础能力 | REQ-001、REQ-002 | MVP |
| 报告与分析闭环 | REQ-003、REQ-006 | MVP |

---

## 4. 范围定义

### 4.1 MVP 必做（P0）

- REQ-001：股票基础信息查询与同步。
- REQ-002：自选分组管理与分组股票维护。
- REQ-003：历史报告列表与详情读取。
- REQ-004：鉴权（登录/刷新/登出）与用户隔离。
- REQ-006：个股分析触发并生成报告。
- NFR-001：统一响应结构与错误码。
- NFR-002：报告访问安全（用户归属+路径约束）。

### 4.2 后续迭代需求（非本期强制交付）

- REQ-008：AI 对话与知识库检索（RAG）。
- REQ-009：持仓管理。
- REQ-010：盯盘告警。
- REQ-011：策略管理与策略执行基础能力。

### 4.3 当前冻结需求（暂不考虑）

- REQ-007：前端市场页与组合页联调。

## 6. 成功指标（MVP）

- P0 需求覆盖率：100%（REQ-001~REQ-004、REQ-006 + NFR-001~002 完成并验收）。
- 核心接口测试：每个 P0 接口至少 1 成功 + 1 失败场景。
- 文档一致性：需求、设计、计划三者不冲突。
- 安全基线：报告读取完成用户归属校验。

---

## 7. 约束与依赖

- 当前数据库基线为 SQLite，后续迁移 PostgreSQL 需单独计划。
- 现有分析管线与新 API 并行，需控制重复模型风险。
- 前端业务页未落地，接口字段需保持稳定。
- REQ-007 当前冻结，不纳入本轮排期与验收。
- 后续迭代需求以 `../02-系统设计/目标态架构蓝图.md` 作为架构约束输入。
