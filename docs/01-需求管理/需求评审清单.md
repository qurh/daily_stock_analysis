# 需求评审清单

**用途：** 在需求进入“待开始（todo）”或“进行中（in-progress）”前进行评审，确保需求具备可执行性。

## 评审前检查

- 需求是否有唯一 ID（`REQ-xxx`/`NFR-xxx`）。
- 需求是否标注阶段归属（`MVP`、`Iteration` 或 `Frozen`）。
- 需求是否明确“范围”和“非目标”。
- 验收标准是否可验证（接口、字段、命令、结果）。
- 是否识别了依赖与风险。

## 评审中检查

- 产品价值是否明确。
- 与当前 MVP 基线是否冲突。
- 与后续迭代路线图是否冲突。
- 若为 Frozen，是否给出明确解冻条件。
- 是否需要新增或修改设计文档。
- 是否需要新增实施计划任务。

## 评审后输出

- 在 `需求池.md` 更新状态与下一步。
- 在 `需求变更记录.md` 记录关键变更。
- 与 `../04-实施计划/` 建立双向链接。
- 指定迭代目标并回写到 `../05-迭代开发记录/`。
