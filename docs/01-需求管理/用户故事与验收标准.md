# 用户故事与验收标准（MVP）

## 用户故事

1. 作为用户，我可以查询和同步股票基础信息。
2. 作为用户，我可以管理自选分组并维护分组股票。
3. 作为用户，我可以查看某只股票的历史报告和报告正文。
4. 作为用户，我可以在后续版本中完成登录鉴权后访问受保护资源。

## 验收标准（当前生效）

- `GET /stocks` 支持 `q/industry/page/page_size/sort`。
- `POST /stocks/sync` 支持从请求体或配置同步股票基础信息。
- `GET/POST/PUT/DELETE /watchlists` 可正常维护分组。
- `POST/DELETE /watchlists/{id}/stocks` 可正常维护分组成分股。
- `GET /stock/{code}/reports`、`GET /reports`、`GET /reports/{id}` 可用。
- 响应结构统一为 `{ code, message, data, request_id }`。

## 待补齐验收标准

- 鉴权与用户隔离。
- 行情列表、个股详情、分析生成。
- 前端页面联调与 E2E。
