## 1. 这是什么

### 1.1 一句话定位

一个 AI 驱动的股票研究与决策辅助系统：把分散的信息变成“可读、可追溯、可复用”的研究结论，并通过告警帮助你及时注意关键变化。

### 1.2 使用方式与关键原则（非常重要）

- 分析报告均为**手动触发**：个股分析、大盘复盘、行业/板块分析都由你点一下生成（不默认自动跑）。  
- 告警是**规则触发**：你定义条件，系统在满足条件时推送提醒。  
- MVP 阶段**单用户优先**：先把个人投研闭环打通，再考虑多用户能力。

### 1.3 你会遇到的痛点

- 信息分散：行情、公告、新闻、研报、财务数据分散在多个渠道，难以形成统一判断。  
- 分析不连续：结论难复用、难回溯，复盘时无法回答“当时为什么这么想”。  
- 执行不及时：关键变化需要频繁盯盘，容易漏掉或错过。

### 1.4 完整闭环与“复盘迭代”的落点

系统覆盖：`数据获取 -> 分析生成 -> 知识沉淀 -> 告警触达 -> 复盘迭代`。  
其中“复盘迭代”的最终落点是：复盘并优化迭代交易策略，推动系统持续自我进化。

### 1.5 不做什么（避免误解）

- 不自动下单、不代替你做交易执行。  
- 不承诺收益；所有输出仅用于研究辅助，需要你自行判断与承担风险。  
- 不以“追热点”为目标，优先追求可解释、可追溯与可复用。

## 2. 你会得到什么（核心产物）

下面的“产物结构”是建议模板，MVP 先做简版，后续逐步补齐。

### 2.1 个股研究报告（手动触发）

- 适用时机：准备建仓/加仓/减仓前；或对持仓标的做例行复盘。  
- 输入：股票代码 + 可选的关注点（例如：基本面/技术面/事件驱动）。  
- 输出结构（建议）：
  - 结论摘要：一句话结论 + 置信度（高/中/低/无法判断）+ 有效期（盘中/日内/周内/长期）  
  - 核心依据：关键数据/事件/指标的引用与解释  
  - 风险清单：最可能推翻结论的 3-5 个风险点  
  - 观察清单：接下来需要盯的指标/公告/价格区间  
  - 行动提示（可选）：只给“建议级别”（信息/分析/行动），不替代你做交易

### 2.2 大盘/市场复盘报告（手动触发）

- 适用时机：收盘后、周末；用于判断市场节奏与仓位管理。  
- 输入：选择指数范围（上证/深成/创业板等）+ 可选的观察维度（情绪/风格/热点/资金）。  
- 输出结构（建议）：市场概览、热点与轮动、情绪与风险、明日/下周观察点、与自身策略的匹配建议。

### 2.3 行业研究报告（手动触发）

- 目标：深入理解行业的景气度、供需/政策/周期、估值与关键公司格局。  
- 输出结构（建议）：行业框架、驱动因素、关键指标、风险与验证点、龙头与受益链条。

### 2.4 板块研究报告（手动触发）

- 目标：深入理解板块（如有色金属、光伏、电力等）的强弱、轮动与资金偏好。  
- 输出结构（建议）：板块强弱、核心成分贡献、资金/情绪、事件驱动、后续观察。

### 2.5 盯盘告警消息（规则触发）

- 适用时机：不想长时间盯盘，但又不希望错过关键变化。  
- 触发类型（建议）：价格/涨跌幅、成交量/换手、关键位突破、事件/公告、板块轮动等。  
- 输出结构（建议）：触发原因、当前状态、历史对比、下一步建议（例如“关注/复核/忽略”）。

### 2.6 复盘条目与反例卡片

- 适用时机：当你的判断被市场验证或证伪时。  
- 内容结构（建议）：当时结论、依据、实际结果、偏差原因、下次如何改（策略/提示词/观察点）。

### 2.7 知识库条目（可检索）

- 目标：把“报告/对话/复盘”变成可检索、可复用的长期资产。  
- 最小能力：标签/分类、全文检索、引用回溯。

## 3. 你怎么用（典型流程）

### 3.1 从 0 到 1（新用户）

1. 同步股票基础信息（代码、名称、上市状态等）。  
2. 建立自选分组（例如：观察池/持仓池/候选池）。  
3. 手动触发个股分析，生成第一版研究报告并保存。  
4. 对关键标的设置盯盘告警（价格区间、关键位、成交量等）。  
5. 盘后手动触发大盘复盘，记录“市场阶段判断”和“明日观察点”。  
6. 根据复盘结果调整策略与观察清单，形成可迭代的研究方法。

### 3.2 日常节奏（建议）

- 盘前：查看昨日日内复盘要点，确认今天的观察清单与告警。  
- 盘中：只处理告警触发的事件（避免无目的盯盘）。  
- 盘后：手动触发大盘复盘 + 关键持仓复盘，沉淀结论与反例。  
- 周末：对行业/板块做一次结构化研究，更新中期假设。

### 3.3 如何读懂报告（避免“AI 说啥信啥”）

- 区分结论类型：事实（可验证）、判断（有依据）、推测（不确定）。  
- 看时效：同一结论在“盘中/日内/周内/长期”含义不同。  
- 看置信度：低置信度时更应输出“观察清单”而非行动建议。  
- 看依据：关键结论必须能回溯到数据、事件或逻辑推理链路。

置信度建议解释（面向读者）：

| 置信度 | 含义 | 使用建议 |
|---|---|---|
| 高 | 多维信息一致，结论相对稳定 | 可以纳入策略，但仍需止损/风控 |
| 中 | 有依据但存在不确定 | 以“观察清单”为主，小仓位验证 |
| 低 | 依据不足或冲突较大 | 仅记录，不建议据此行动 |
| 无法判断 | 信息不足或互相矛盾 | 暂停结论，等待关键验证点 |

## 5. MVP 与迭代规划（单用户优先）

### 5.1 MVP 的定义（你能“闭环用起来”）

MVP 的目标不是“功能最多”，而是你能用它完成一次完整闭环：

- 手动触发个股分析 -> 生成并保存报告 -> 可回看/可检索 -> 设置告警 -> 盘后复盘 -> 复盘驱动策略/方法迭代。

### 5.2 MVP（V1）建议范围

- P0（必须）：REQ-001, REQ-002, REQ-003, REQ-006, REQ-021  
- P1（建议纳入 V1 或紧随其后）：REQ-012, REQ-013, REQ-014, REQ-015, REQ-019, REQ-020  

说明：V1 中所有“分析类报告”均为手动触发；是否增加定时任务属于后续可选能力。

### 5.3 后续迭代方向（P2/P3）

- 交易闭环增强：持仓管理、盯盘告警、策略管理（REQ-009, REQ-010, REQ-011）。  
- 认知与可靠性增强：输出风格/责任边界、冲突信息与置信度机制（REQ-016, REQ-017）。  
- 市场研究深化：行业与板块分析的维度补齐与可比性（REQ-022, REQ-023）。  
- （如需要）多用户与更严格的权限隔离（REQ-004 升级）。

## 附录 B：逐项验收要点（简版）

> 这里给研发与验收使用，个人投资者可跳过。

### REQ-001 股票基础信息查询与同步

- 用户故事：作为投资者，我希望系统能同步并查询股票基础信息，以便自选、分析与报告生成有可靠底座。  
- 验收要点：
  - 可按代码/名称查询股票基础信息（至少包含代码、名称、上市状态等）  
  - 支持定期/按需同步更新，并记录更新时间  
  - 同一股票在不同数据源下的关键字段口径一致（或有明确说明）

### REQ-002 自选分组与分组股票维护

- 用户故事：作为投资者，我希望用分组管理自选股票，以便按策略/阶段组织关注池。  
- 验收要点：
  - 支持创建/重命名/删除分组，分组下可增删股票  
  - 支持快速切换分组查看与搜索定位  
  - 分组与股票关系可持久化保存

### REQ-003 历史报告列表与详情读取

- 用户故事：作为投资者，我希望能快速回看历史报告与当时结论，以便复盘与追溯。  
- 验收要点：
  - 支持按股票/日期/类型检索历史报告  
  - 报告详情可打开阅读，包含生成时间与关键结论  
  - 报告内容可复制/导出（MVP 可先支持复制）

### REQ-004 基础访问控制（单用户）

- 用户故事：作为单用户，我希望系统有基础访问控制，以免服务暴露导致他人访问我的数据。  
- 验收要点：
  - Web UI/接口默认需要访问凭证（例如访问密钥/登录）  
  - 支持配置与轮换凭证（MVP 可先支持手动修改）  
  - 访问失败有明确提示，不泄露敏感信息

### REQ-006 个股分析触发与报告生成

- 用户故事：作为投资者，我希望手动触发某只股票的分析，生成结构化报告并保存，以便后续复盘与追溯。  
- 验收要点：
  - 支持输入股票代码一键触发，生成报告并落库  
  - 报告包含：结论摘要、依据、风险清单、观察清单、有效期与置信度  
  - 报告可在历史列表中检索与打开，并记录使用的数据时间与来源

### REQ-008 AI 对话与知识检索

- 用户故事：作为投资者，我希望能用对话方式提问，并能引用我过去的报告与笔记，以便快速得到有根据的回答。  
- 验收要点：
  - 支持多轮对话提问与历史会话查看  
  - 回答能引用历史报告/知识库条目（至少给出引用标题与时间）  
  - 支持将对话中的高价值结论一键沉淀为知识条目（MVP 可先支持手动保存）

### REQ-009 持仓管理

- 用户故事：作为投资者，我希望记录持仓与交易，以便从“账户视角”复盘决策质量。  
- 验收要点：
  - 支持录入持仓与交易记录（买入/卖出、价格、数量、时间）  
  - 支持按标的/时间查看持仓变化与盈亏（口径明确）  
  - 支持与报告/复盘建立关联（MVP 可先做弱关联：按股票自动关联）

### REQ-010 盯盘告警

- 用户故事：作为投资者，我希望设置盯盘规则并收到告警，以便不盯盘也能及时处理关键变化。  
- 验收要点：
  - 支持配置告警规则（至少覆盖价格/涨跌幅/成交量或换手其中两类）  
  - 告警消息包含触发原因与当前状态，支持查看历史告警  
  - 支持防打扰（去重/冷却时间/分级）以减少噪音

### REQ-011 策略管理与策略执行

- 用户故事：作为投资者，我希望把自己的交易策略沉淀成可复用的规则，并能评估策略表现。  
- 验收要点：
  - 支持创建/编辑/版本化策略（至少包含策略说明、入场/出场原则）  
  - 支持记录策略信号与结果，形成复盘数据  
  - 支持策略与告警/报告联动（MVP 可先只做策略定义与记录）

### REQ-012 大盘复盘与市场洞察

- 用户故事：作为投资者，我希望在收盘后手动生成大盘复盘，判断市场节奏与风险。  
- 验收要点：
  - 支持手动触发并生成结构化复盘报告  
  - 报告包含：市场概览、热点与轮动、情绪与风险、观察点  
  - 报告可落库并可回看，记录复盘日期与数据来源

### REQ-013 新闻搜索与公告解析

- 用户故事：作为投资者，我希望搜索新闻并解析公告重点，以便快速判断事件影响。  
- 验收要点：
  - 支持按股票/关键词搜索新闻与公告  
  - 支持提取摘要与关键点（影响方向、风险点、待验证点）  
  - 支持将重要事件一键关联到个股报告/复盘（MVP 可先支持复制引用）

### REQ-014 知识库管理与沉淀

- 用户故事：作为投资者，我希望把报告、复盘与对话沉淀为可检索知识库，以便长期复用。  
- 验收要点：
  - 支持创建/编辑知识条目，支持标签/分类  
  - 支持全文检索与按标签筛选  
  - 支持从报告/对话一键生成知识条目（MVP 可先支持手动复制创建）

### REQ-015 AI 分析环节与提示词（Prompt）管理

- 用户故事：作为投资者，我希望不同分析任务有可调整的提示词模板，以便逐步优化报告质量。  
- 验收要点：
  - 先梳理系统中的“分析任务清单”（个股/大盘/行业/板块/公告等）  
  - 每个任务可配置提示词模板，支持版本与回退（MVP 可先支持版本号+历史查看）  
  - 提示词修改能在下一次生成报告时生效，并可追溯使用了哪个版本

### REQ-016 输出风格与责任边界配置

- 用户故事：作为投资者，我希望系统明确输出级别与风险提示，避免把分析当成确定性结论。  
- 验收要点：
  - 支持配置输出级别（信息/分析/行动建议）  
  - 对高风险场景强制输出风险提示与免责声明  
  - 允许输出“不确定/无法判断”，并给出验证点

### REQ-017 冲突信息与置信度机制

- 用户故事：作为投资者，我希望当信息冲突时系统能说明矛盾并给出置信度与应对策略。  
- 验收要点：
  - 能识别常见冲突（指标冲突/周期冲突/来源冲突/逻辑冲突）  
  - 输出置信度（高/中/低/无法判断）并解释原因  
  - 给出下一步验证点或多情景输出

### REQ-019 运行模式与任务流程

- 用户故事：作为单用户，我希望系统运行模式清晰、任务状态可见，以便稳定使用与排查问题。  
- 验收要点：
  - 支持不同运行入口/模式（例如仅 Web、仅分析、分析+通知等）  
  - 任务执行过程可追踪（开始/结束/失败原因）  
  - 失败可重试，不会造成重复数据或混乱

### REQ-020 多渠道消息通知能力

- 用户故事：作为投资者，我希望告警与关键结果能通过我常用渠道送达，以便及时处理。  
- 验收要点：
  - 支持至少 2 种通知渠道（以现有实现为准）  
  - 支持消息模板与重试机制，避免丢消息  
  - 支持按类型开关通知（告警/报告完成/失败等）

### REQ-021 多数据源容错与口径一致

- 用户故事：作为投资者，我希望数据源波动时系统仍能正常生成报告，且关键口径一致。  
- 验收要点：
  - 单一数据源失败时可降级切换或给出明确提示  
  - 关键字段口径一致（或提供口径说明，避免“同名不同义”）  
  - 支持基础的数据质量检查与异常提示（MVP 可先做关键字段缺失检测）

### REQ-022 行业分析

- 用户故事：作为投资者，我希望手动生成行业研究报告，理解行业景气与驱动因素。  
- 验收要点：
  - 支持选择行业并生成结构化行业报告  
  - 报告包含：行业框架、核心驱动、关键指标、风险与验证点  
  - 支持与行业内核心公司/个股报告建立关联（MVP 可先做引用列表）

### REQ-023 板块分析

- 用户故事：作为投资者，我希望手动生成板块研究报告，理解板块强弱与轮动。  
- 验收要点：
  - 支持选择板块并生成结构化板块报告  
  - 报告包含：板块强弱、核心成分贡献、资金/情绪、事件驱动  
  - 支持对比不同板块的关键结论（MVP 可先支持并列查看）

## 附录 C：质量底线（投资者关心）

- 数据可信：重要字段口径一致、来源可见、异常可提示。  
- 可追溯：关键结论能回看“当时使用的数据/事件”。  
- 稳定可用：日常使用不因单一数据源波动而频繁失败。  
- 安全底线：API Key 等敏感信息不泄露；单用户场景避免公开暴露。  
- 体验流畅：生成与检索在可接受时间内完成。
