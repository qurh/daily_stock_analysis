# 需求池

**最后更新：** 2026-02-06

## 1. 入池规则

- 必须包含：需求标题、业务价值、影响范围、验收口径。
- 必须分配 ID：`REQ-xxx` 或 `NFR-xxx`。
- 必须标注优先级：`P0/P1/P2/P3`。
- 未经评审不得进入“进行中（in-progress）”。
- 状态统一使用：已完成（done）、进行中（in-progress）、待开始（todo）、待排期（backlog）、暂缓（on-hold）、候选（candidate）。
