# 需求管理目录说明

## 目标

本目录用于维护需求分析、需求治理、排期看板、验收口径与变更记录，保证需求从提出到交付全链路可追踪。

## 单一事实源

- 需求内容基线：`需求分析.md`
- 需求流程基线：`需求管理.md`

## 文档清单

- `需求分析.md`：需求分析母文档（背景、用户需求、REQ/NFR、验收基线）。
- `需求管理.md`：需求治理文档（状态、流程、评审、变更、冻结解冻规则）。
- `需求池.md`：需求排期与状态看板。
- `用户故事与验收标准.md`：按需求编号定义验收口径。
- `需求变更记录.md`：需求变更审计记录。
- `需求评审清单.md`：需求评审门禁清单。

## 建议使用顺序

1. 先看 `需求分析.md` 明确“做什么”。
2. 再看 `需求管理.md` 明确“怎么管理”。
3. 进入 `需求池.md` 与 `用户故事与验收标准.md` 做迭代执行。
4. 发生变更时，更新 `需求变更记录.md` 并回写相关文档。
