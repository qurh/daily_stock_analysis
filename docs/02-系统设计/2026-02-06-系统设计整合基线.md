# 2026-02-06 系统设计整合基线

**状态：** 生效  
**负责人：** 项目组  
**最后更新：** 2026-02-06

---

## 1. 设计目标

在不打断现有开发节奏的前提下，建立“目标态架构”和“当前实现基线”的双轨描述：
- 目标态：FastAPI + PostgreSQL + Alembic + JWT。
- 当前态：FastAPI + SQLite + 分阶段补齐鉴权与业务 API。

---

## 2. 当前生效架构

- 后端入口：`backend/app/main.py`
- 路由聚合：`backend/app/api/router.py`
- 现有核心模块：`stocks`、`watchlists`、`reports`
- 数据层：SQLAlchemy（SQLite）

---

## 3. 数据与接口基线

- 已实现模型：`stocks`、`watchlists`、`watchlist_stocks`、`analysis_reports`
- 已实现接口：
  - `/stocks`、`/stocks/sync`
  - `/watchlists` 及其子资源
  - `/stock/{code}/reports`、`/reports`、`/reports/{id}`

---

## 4. 目标态与差异

| 主题 | 目标态 | 当前态 | 补齐阶段 |
|---|---|---|---|
| 鉴权 | JWT + Refresh 轮换 | 未实现 | Phase 2 |
| 多用户隔离 | 用户归属严格校验 | 部分接口固定 user_id=1 | Phase 2 |
| 行情接口 | `/market/list` + `/stock/{code}/detail` | 未实现 | Phase 2 |
| 分析生成 | `/stock/{code}/analyze` | 未实现 | Phase 2 |
| 数据库 | PostgreSQL + Alembic | SQLite | Phase 4 前评估 |

---

## 5. 设计约束

- 所有新增接口必须遵循统一响应结构。
- 安全约束必须前置：报告读取、路径访问、用户归属。
- 先保证 MVP 闭环可用，再扩展高级模块。
