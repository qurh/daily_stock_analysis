# 系统设计基线

## 1. 当前技术基线（2026-02-06）

- 后端：FastAPI + SQLAlchemy。
- 数据库：SQLite（当前生效），PostgreSQL 为后续迁移目标。
- 前端：Next.js 脚手架状态，尚未进入业务页面实现。
- 报告：Markdown 文件 + 元信息入库。

## 2. MVP 范围（当前生效）

- 股票基础信息
- 自选分组
- 历史报告
- 行情列表（待实现）
- 个股详情（待实现）
- 个股分析触发（待实现）

## 3. 与历史设计差异

- 历史文档中包含 AI 对话、知识库、持仓、盯盘等模块，当前不纳入 MVP 必选交付。
- 历史文档中 PostgreSQL + Alembic 为目标态，当前迭代采用 SQLite 快速落地。

## 4. 迁移原则

- 阶段内允许“目标态未落地”，但必须写明“当前基线”与“目标态”差异。
- 差异必须在实施计划中有明确补齐任务，不允许长期隐性漂移。
