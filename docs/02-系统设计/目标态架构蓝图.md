# 目标态架构蓝图（后MVP）

**状态：** 生效（目标态参考）  
**最后更新：** 2026-02-06

---

## 1. 定位

本文件整合历史《架构设计文档》《架构设计详细分析报告》的长期目标内容，作为 MVP 完成后的演进蓝图。

该蓝图不改变当前执行基线，当前研发仍以 `2026-02-06-系统设计整合基线.md` 为准。

---

## 2. 目标态能力版图

- 市场数据与个股分析服务
- 大盘复盘与报告沉淀
- 知识库/RAG 与对话系统
- 持仓、盯盘、策略管理
- 认知状态、反例库、结论生命周期

---

## 3. 目标态技术方向

- 后端：FastAPI + PostgreSQL + Alembic
- 数据：结构化库 + 文档存储 + 向量检索
- 前端：Next.js + 组件化设计系统
- 运维：容器化部署 + 可观测性 + 安全审计

---

## 4. 分阶段演进建议

1. MVP 收敛阶段：完成鉴权、核心 API、前端核心页。
2. 能力扩展阶段：接入知识库和对话。
3. 决策增强阶段：引入认知状态与反馈闭环。
4. 平台化阶段：完善策略与监控体系。
