# ADR（架构决策记录）

## ADR-001：MVP 阶段先以 SQLite 交付 API 能力

- 状态：accepted
- 日期：2026-02-06
- 决策：当前阶段优先交付业务 API，数据库采用 SQLite；在后续阶段迁移到 PostgreSQL + Alembic。
- 原因：当前仓库已有 SQLite 运行基线，迁移成本和不确定性高。
- 影响：需要在实施计划中明确迁移步骤与验收标准。

## ADR-002：将“目标态架构文档”和“当前生效基线”分离

- 状态：accepted
- 日期：2026-02-06
- 决策：系统设计类文档必须同时给出目标态与当前态，并标明差异。
- 原因：减少误读，防止“文档写了但并未实现”。
- 影响：新增基线文档维护成本，但提升执行一致性。
