# 文档体系总览

## 1. 目标

本目录用于建立可持续的研发文档体系，确保需求、设计、实施、交付和复盘全链路可追踪。

核心目标：
- 单一事实源：减少多份文档相互冲突。
- 可执行：每个计划都能直接指导开发。
- 可追溯：任一功能可回溯到需求、设计、提交和验收。
- 可迭代：每次迭代有记录、有总结、有改进项。

## 2. 目录结构

- `00-文档治理/`：文档规范、命名规则、术语表。
- `01-需求管理/`：需求分析、需求管理、需求池、需求变更、用户故事与验收标准。
- `02-系统设计/`：架构设计、数据模型、API 基线、ADR。
- `03-工程化/`：开发规范、测试策略、发布与回滚。
- `04-实施计划/`：路线图、里程碑、可执行实施计划。
- `05-迭代开发记录/`：每次迭代的进展、问题、决策与结论。
- `templates/`：需求、设计、计划、迭代记录模板。
- `99-历史文档/`：历史文档索引与迁移说明。
