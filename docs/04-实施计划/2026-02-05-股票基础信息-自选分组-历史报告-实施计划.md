# 2026-02-05 股票基础信息-自选分组-历史报告-实施计划

**状态：** 部分完成（持续更新）  
**最后更新：** 2026-02-06  
**范围：** MVP 后端 API 的三项核心能力及其后续补齐项

---

## 1. 目标

构建并稳定以下后端能力：
- 股票基础信息：查询与同步
- 自选分组：分组维护与成分股维护
- 历史报告：个股报告列表、全局报告列表、报告详情

---

## 2. 当前进度

### 2.1 已完成

- `GET /stocks`
- `POST /stocks/sync`
- `GET/POST/PUT/DELETE /watchlists`
- `POST/DELETE /watchlists/{watchlist_id}/stocks`
- `GET /stock/{code}/reports`
- `GET /reports`
- `GET /reports/{report_id}`
- 关键接口测试已补齐并通过（3 个场景）

### 2.2 未完成

- 鉴权与用户隔离
- 报告访问权限校验
- 路径安全限制与更严格校验
- 行情列表/个股详情/分析触发 API

---

## 3. 分批执行计划

### 批次 A（已完成）

- 建立 FastAPI 入口与统一响应结构
- 建立 stocks/watchlists/analysis_reports 数据模型
- 完成三组核心 API 与基础测试

### 批次 B（待执行）

- 引入用户与鉴权模型
- 实现 `POST /auth/login`、`POST /auth/refresh`、`POST /auth/logout`
- 将 watchlist/report 查询从固定用户改为鉴权用户

### 批次 C（待执行）

- 实现 `GET /market/list`
- 实现 `GET /stock/{code}/detail`
- 实现 `POST /stock/{code}/analyze`
- 补齐对应集成测试

### 批次 D（待执行）

- 安全补齐：报告路径白名单、用户归属校验
- 性能与稳定性验证
- 与前端联调并完成验收

---

## 4. 验收标准

- 接口响应结构统一：`{ code, message, data, request_id }`
- 每个接口具备成功与关键失败场景测试
- 文档、实现、测试三者一致

---

## 5. 风险与对策

- 风险：历史文档口径冲突导致重复实现。
- 对策：以本计划与 `02-系统设计/数据模型与API基线.md` 为当前执行基线。

- 风险：鉴权晚接入导致后续改动范围扩大。
- 对策：将批次 B 作为下一优先级任务。
