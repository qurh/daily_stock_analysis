## 使用规则

- 每个计划必须有状态字段（未开始/进行中/已完成/阻塞）。
- 每次开发结束后更新完成度与下一步。
- 计划文档是执行基准，不以聊天记录代替。
- 与当前执行冲突时，以 `2026-02-06-从0到1-项目规划.md` 为准。
- 对于 `01-需求管理/需求池.md` 中 `on-hold` 的需求，计划中仅保留状态说明，不进入当前里程碑验收。
