# 后端 API 与数据模型设计（MVP）

**项目名称：** daily_stock_analysis  
**版本：** v1.0  
**日期：** 2026-02-05  
**状态：** 设计定稿  

---

## 1. 技术选型与目标

- 数据库：PostgreSQL  
- ORM/迁移：SQLAlchemy + Alembic  
- 鉴权：用户名/密码 + JWT（Access 1 小时、Refresh 7 天，Refresh 轮换）  
- 密码哈希：bcrypt  
- 报告存储：Markdown 正文落文件系统，数据库仅存路径与元信息  
- 报告路径：`backend/reports/YYYY-MM-DD/<code>/HHMMSS.md`  

目标是支撑：行情列表、个股详情、个股分析生成、报告历史、分组自选与多用户鉴权。

---

## 2. 数据模型（核心表）

### 2.1 用户与鉴权
- `users`：`id`、`username`、`password_hash`、`status`、`created_at`  
- `refresh_tokens`：`id`、`user_id`、`token_hash`、`expires_at`、`revoked_at`、`rotation_parent_id`  
  - Refresh Token 轮换：刷新后旧 token 立即作废，记录关联链路便于审计  

### 2.2 股票基础信息
- `stocks`：`code`、`name`、`industry`、`market`、`is_active`、`updated_at`  
  - 定期同步（后台任务），用于行情列表与分组展示  

### 2.3 自选与分组
- `watchlists`：`id`、`user_id`、`name`、`sort_order`、`created_at`  
- `watchlist_stocks`：`watchlist_id`、`code`、`sort_order`、`added_at`  

### 2.4 行情与指标
- `stock_quotes`（分钟级快照）：`code`、`ts`、`price`、`pct_chg`、`volume`、`amount`、`turnover`、`volume_ratio`、`net_inflow`…  
- `stock_daily`（日线指标）：`code`、`date`、`open`、`high`、`low`、`close`、`ma5/10/20`…  

### 2.5 个股分析报告
- `analysis_reports`：  
  - `id`、`user_id`、`code`、`report_type`、`report_date`  
  - `generated_at`、`markdown_path`、`summary_json`、`tags`  
  - `model`、`source`、`status`  
  
说明：  
1) 每次“个股分析”都会生成一条记录，同一天允许多条；  
2) Markdown 正文以 `HHMMSS.md` 命名，保留全部历史版本；  
3) `summary_json` 保存结构化指标摘要，便于列表展示与检索。  

### 2.6 关键索引
- `analysis_reports(code, generated_at DESC)`  
 - `watchlist_stocks(watchlist_id, code)`  
 - `stock_quotes(code, ts)`  
 - `stocks(code)` 唯一

## 3. API 设计（MVP）

### 3.1 鉴权
- `POST /auth/login` → access + refresh  
- `POST /auth/refresh` → 轮换 refresh，返回新 access  
- `POST /auth/logout` → 撤销 refresh  

### 3.2 基础数据
- `GET /stocks`（按 code/name/industry 查询）  
- `POST /stocks/sync`（后台同步任务）  

### 3.3 自选与分组
- `GET /watchlists`  
- `POST /watchlists`  
- `PUT /watchlists/{id}`  
- `DELETE /watchlists/{id}`  
- `POST /watchlists/{id}/stocks`  
- `DELETE /watchlists/{id}/stocks/{code}`  

### 3.4 行情与详情
- `GET /market/list`（支持分组筛选/排序）  
- `GET /stock/{code}/detail`（行情 + 日线 + 摘要）  

### 3.5 分析与报告
- `POST /stock/{code}/analyze`（生成最新分析，返回报告 ID）  
- `GET /stock/{code}/reports`（该股历史）  
- `GET /reports`（全局历史，支持日期/代码筛选）  
- `GET /reports/{id}`（渲染 Markdown）

## 4. 错误处理与安全策略

统一响应结构：`{ code, message, data, request_id }`。  
鉴权失败 `401`，无权限 `403`，资源不存在 `404`，参数错误 `422`，内部错误 `500`。  
日志脱敏（API Key、token 等），仅保留 `request_id` 便于排障。  
报告读取需校验用户归属；Markdown 路径禁止直接外链访问。  
外部数据源故障返回 `partial=true` 并标记 `data_source`。  
LLM 失败时降级为模板输出，`source=template`、`status=degraded`。  
统一时区：`Asia/Shanghai`（UTC+8）。
