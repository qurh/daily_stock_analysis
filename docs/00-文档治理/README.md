# 文档治理

## 1. 文档分级

- L1：方向文档（需求范围、系统蓝图）。
- L2：执行文档（实施计划、工程流程）。
- L3：记录文档（迭代日志、问题清单）。

## 2. 命名规范

- 日期前缀：`YYYY-MM-DD-主题.md`。
- 计划文档：`...-实施计划.md`。
- 记录文档：`...-迭代记录.md`。
- 设计文档：`...-设计.md`。

## 3. 状态字段

每份核心文档建议包含：
- `状态`：草稿 / 评审中 / 生效 / 归档
- `负责人`
- `最后更新`

## 4. 版本与变更

- 小改动：直接改文并更新“最后更新”。
- 大改动：在 `01-需求管理/需求变更记录.md` 写明变更原因、影响范围、回滚方式。

## 5. 质量要求

- 避免口号式描述，优先写“可执行步骤 + 验证标准”。
- 接口和字段必须与代码一致，不一致时以“实施计划”状态说明。
- 所有时间统一使用 `Asia/Shanghai`。
